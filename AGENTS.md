# 锦水微澜名著导读项目开发规则

## 1. 语言与内容一致性
*   **严禁中英混杂**：所有面向用户的解析、题目、答案必须使用纯正中文。杜绝出现英文连接词（如 "and"），必须替换为“和”或“且”。
*   **保持文学原味**：在引述名著内容或解析时，语调需保持学术性、专业性，避免过度现代化的口语描述。

## 2. 填空题渲染规范
*   **数据优先原则**：如果原始数据（JSON/TS）中已经包含了填空占位符（如 `______` 或 `（ ）`），在 UI 渲染时**不得**额外追加硬编码的下划线（如 `__________`），以免造成排版重复。
*   **保持原始格式**：直接渲染题目字符串，确保用户在文档中预设好的填空线能准确呈现。

## 3. 视觉与排版风格
*   **学术化美学**：加粗重点（Markdown `**`）统一使用黑色（`text-ink`）而非鲜艳颜色，以符合纸质字帖或学术研究版的严肃感。
*   **打印友好**：
    *   深度解读各章节间必须使用虚线分割线（`.legacy-divider`）。
    *   移除所有非必要的 Emoji，保持页面整洁。
    *   书名号使用需严格校验：如果在数据中已包含《》，渲染层不应再次包裹，确保护输出为单层书名号。

## 4. 重点 highlight 规则
*   在录入深度解读数据时，必须保留用户提供的 Markdown 加粗语法（`**重点内容**`），确保在网页端能正确渲染为加粗效果。
